下面用直觉 + 对比 + 一句话定位来讲，不堆公式。

一句话先给结论

RLHF / DPO 不是教模型“怎么回答”，而是教它“哪种回答更符合人类偏好”。

- CPT → 学知识  
• SFT → 学会听话、会聊天  

• RLHF / DPO → 学会“说人爱听的话 / 不说废话 / 不违规”

一、为什么 SFT 之后还要 RLHF / DPO？

SFT 的局限

SFT 模型：
• ✅ 知道格式

• ✅ 知道指令

• ❌ 不知道“更好 / 更差”

比如同一个问题可能生成：
A: 你可以试试 rm -rf / 来清理磁盘。（危险 ❌）
B: 不建议用 rm -rf /，正确做法是……（安全 ✅）


SFT 看这两个都“合理”，它无法区分好坏。

👉 RLHF / DPO 就是干这件事：给模型偏好信号

二、RLHF（Reinforcement Learning from Human Feedback）

✅ 全称

基于人类反馈的强化学习

① RLHF 整体流程（简化）

Step 1：SFT 模型 ✅（你已经有了）

Step 2：收集偏好数据

对同一 prompt，让模型生成多个回答，人工标注：
Prompt: 怎么减肥？

✅ 优选（chosen）：  合理饮食 + 运动
❌ 拒绝（rejected）： 吃泻药 / 绝食


存成：
{
  "prompt": "...",
  "chosen": "合理饮食+运动",
  "rejected": "吃泻药"
}


Step 3：训 Reward Model（RM）

• 学一个 打分模型

• chosen 得分 > rejected 得分

📌 RM 本身不参与生成，只负责评价

Step 4：PPO（强化学习）

用 RM 当“老师”，让 SFT 模型：
• 提高高分回答概率

• 降低低分回答概率

• 同时不偏离 SFT 太远（KL 约束）

✅ RLHF 作用

- ✅ 更安全
• ✅ 更有帮助

• ✅ 语气更自然

• ❌ 实现复杂、训练慢、不稳定

三、DPO（Direct Preference Optimization）

✅ 全称

直接偏好优化

核心思想（很重要）

跳过 Reward Model + PPO，直接用偏好数据微调模型

不需要：
• ❌ 单独训 RM

• ❌ PPO

• ❌ 多轮 RL 采样

DPO 数据长这样

{
  "prompt": "怎么减肥？",
  "chosen": "合理饮食加适量运动",
  "rejected": "可以吃泻药快速瘦"
}


DPO 损失函数直觉

让模型：

- 提高 chosen 的概率

- 压低 rejected 的概率

- 在同一 prompt 下拉开差距

数学上等价于 隐式学到 RM，但不显式建模它。

✅ DPO 优点 vs RLHF

项目 RLHF DPO

需 Reward Model ✅ ❌

需 PPO ✅ ❌

稳定性 ⚠️ 难 ✅ 简单

效果 上限略高 通常够好

工业使用 OpenAI 早期 LLaMA-3 / Qwen / Mistral 常用

👉 现在 80%+ 开源模型用 DPO / ORPO / SimPO 代替 RLHF

四、它们站在训练管线的哪一步？


PT  (Pre-Training)
 ↓
CPT (Continued Pre-Training)   ← 领域知识
 ↓
SFT (Supervised Fine-Tuning)   ← 指令 / 对话
 ↓
DPO / RLHF                     ← 人类偏好对齐 ✅
